Setup Process

Follow these steps to set up the email processor to work with FakeMail:

1
Create a Webhook URL with Ngrok

To expose your local webhook endpoint to FakeMail, you need to create a publicly accessible URL using Ngrok.

  1. Install Ngrok if you haven't already
  2. Run the following command in your terminal:
ngrok http 8000

This will create a tunnel to your local server. Copy the https URL provided by Ngrok.

2
Configure the Webhook URL

Enter your Ngrok URL + "/webhook" in the form below:

This is the URL that FakeMail will use to notify your application when new emails arrive.

Test the Integration

To test that everything is working correctly:

  1. Go to the FakeMail Simulator
  2. Send a test email
  3. Check the Processed Emails page to see if the email was processed

Troubleshooting

Common Issues

  • Webhook not receiving notifications: Ensure your Ngrok tunnel is active and the URL is correct.
  • Emails not being processed: Check the system logs for errors.
  • Classification not working: Ensure the FakeMail API URL is correctly configured.
